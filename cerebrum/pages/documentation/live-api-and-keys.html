Live API and keys

Live mode calls the provider you select from the browser. NERVE’s scoring stays in-page as JavaScript; only the chat completion (or messages) request goes to the provider.

What the model sees

If Context is empty, the API request body uses the Prompt only. If context is present, the app sends one user message: a labeled grounding block, then the user request—so the model can condition on your facts. NERVE still evaluates using the separate Prompt, Context, and Response fields, not a hidden wrapper string.

Confidence: logprobs vs proxy

  • OpenAI / Groq — Requests may include logprobs; if returned, Confidence(t) is derived from token logprobs.
  • Groq — If a logprobs request fails, the app may retry without logprobs; Confidence falls back to the linguistic proxy.
  • Anthropic / Gemini — No logprobs in the integration; linguistic confidence is used for Confidence(t).

Browser and CORS

Calls use fetch to the provider’s public API. Whether a key works in-browser depends on the provider’s CORS policy and, for Anthropic, browser-specific headers. If requests fail, check those policies and any origin restrictions on the provider side.
