What is NERVE?

NERVE (Neural Epistemic Reliability and Validation Engine) scores model outputs against the grounding you provide. Everything runs locally in the browser: no weights shipped, no server-side scoring step.

Canonical specification snapshot

    Core quantities

    All of these are derived from the prompt, optional context, and model output (and, when available, token logprobs):

    • Φ_e(t) — How well the output is aligned with the supplied context (lexical / substantive overlap; higher when the answer and context overlap).
    • CI(t) — Quality and presence of grounding. Empty context uses a low skeptical prior; richer context raises CI.
    • Φ_e adjusted = Φ_e × CI — Primary admissibility signal, compared to ε (epsilon). If below ε, the governed mode is ABSTAIN (treated as no admissible answer in the tool).
    • ESR(t) — Epistemic support ratio: how much of the output is tied to the grounding text.
    • Confidence(t) — From token logprobs on APIs that return them; otherwise a linguistic confidence heuristic (weaker for cross-model comparison).
    • EFCG(t) = Confidence(t) − ESR(t) — The epistemic fidelity calibration gap. Large positive values read as overconfidence relative to support.

    Labels (CORRECT / HALLUCINATION / UNCERTAIN)

    These are your ground-truth flags for experiments (e.g. AUC-ROC). They do not change the measured signals.

    Modes (governance)

    • ASSERT — Admissible (Φ_e adjusted ≥ ε) and EFCG within tolerance.
    • QUALIFY — Admissible but EFCG elevated; output may be prefixed with a qualification.
    • ABSTAIN — Not admissible: Φ_e adjusted < ε.
